[ ] 4. ADD SHARED TEMPLATE APPROACH
    Every gallery page repeats the same boilerplate:
      - <head> with meta tags, CSS link, preloads
      - Bio sidebar with placeholder image and nav links
      - Scrollbox container structure
      - <script> tag at bottom

    Options (from simple to advanced):

    Option A — HTML includes via JavaScript:
      Extract shared header/footer into partials.
      Load with fetch() on page load.
      Pro: No build step. Con: Flash of unstyled content.

    Option B — Static site generator (recommended):
      Use 11ty (eleventy), a lightweight static site
      generator. Write content in HTML or Markdown,
      define a shared layout template, and 11ty builds
      the final HTML files.
      Pro: Clean separation of content and layout.
      Con: Adds a build step (npm install + npx eleventy).

    Option C — Simple build script:
      Write a Python or Node script that combines a
      shared template with per-page content files.
      Pro: No new dependencies. Con: Custom tooling.

    This is optional and only worthwhile if the site will
    continue to grow with new gallery pages.
